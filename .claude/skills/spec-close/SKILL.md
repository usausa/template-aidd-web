# クローズ蒸留(spec を移して消す)

> 原則: **復元できないものを移してから消す**。spec は一時物で、恒久の受け皿は ADR / glossary / テスト名。

## 手順

1. 対象の `work/SPEC-*.md` / `work/PLAN-*.md` を読み、各記述を 3 分類する:
   - **移す**: 決定と理由 → `docs/adr/`(`/adr`)/ 用語 → `docs/glossary.md` / 後から効く背景 → 関連 ADR の「背景」
   - **反映確認**: 受け入れ条件(Given / When / Then)が**テスト名**に反映されているか。漏れがあればテストを足す
   - **捨てる**: 実装詳細・手順・チェックリスト = コードと git 履歴で分かる
2. PLAN のチェックリストが全て `- [x]` であることを確認する(残があれば未完了 = 消さない)。
3. `work/` の該当ファイルを削除する。

## 判断基準(1 行)

> この行は**コード・テスト・ADR・glossary のどれかから復元できるか?** できないなら移してから消す。
